# サイトを公開するまでの手順【非エンジニア向け】

このガイドは、プログラミング経験がない方でも、Sota.Nのポートフォリオサイトを公開できるように、できるだけわかりやすく説明しています。

---

## 🎯 ゴール

あなたのポートフォリオサイトを、誰でもインターネット上で見られるようにする。

例：`https://あなたのサイト名.vercel.app` というURLでアクセスできるようになります。

---

## 📋 事前準備（必要なもの）

### 1. パソコン
- Mac または Windows

### 2. インターネット接続

### 3. アカウント登録（無料）
以下のサービスに登録する必要があります：

- **GitHub**（ギットハブ）- コードを保存する場所
  - URL: https://github.com
  - 無料プラン: ⭕ あります
  
- **Vercel**（バーセル）- サイトを公開するサービス
  - URL: https://vercel.com
  - 無料プラン: ⭕ あります（個人サイトには十分）

**💡 ポイント**: どちらも無料で使えます！クレジットカードの登録も不要です。

---

## 🚀 公開までの全体の流れ（5ステップ）

```
【現在地】
設計書だけがある状態
    ↓
Step 1: 開発環境を整える（パソコンに必要なソフトをインストール）
    ↓
Step 2: サイトを作る（コードを書く）
    ↓
Step 3: GitHubに保存する
    ↓
Step 4: Vercelで公開する
    ↓
【完成！】
世界中からアクセスできるサイトが完成
```

**所要時間**: 初めての方で4週間程度（1日1〜2時間作業した場合）

## Step 1: 開発環境を整える

### 1-1. ソフトウェアのインストール

#### ① Node.js（ノードジェーエス）をインストール

**Node.jsとは？**  
JavaScriptというプログラミング言語を動かすためのソフトです。

**インストール方法**:

1. https://nodejs.org/ja にアクセス
2. 「推奨版」と書かれた緑のボタンをクリック
3. ダウンロードしたファイルをダブルクリック
4. 指示に従ってインストール

**確認方法**（ターミナル/コマンドプロンプトで）:
```bash
node -v
```
→ `v18.0.0` のようなバージョン番号が表示されればOK

#### ② Git（ギット）をインストール

**Gitとは？**  
コードの変更履歴を管理するためのツールです。

**インストール方法**:

- **Macの場合**: 
  1. ターミナルを開く（アプリケーション → ユーティリティ → ターミナル）
  2. 以下を入力してEnter:
     ```bash
     git --version
     ```
  3. 自動的にインストールが始まります（すでにインストール済みならバージョンが表示されます）

- **Windowsの場合**:
  1. https://git-scm.com/download/win にアクセス
  2. 自動的にダウンロードが始まります
  3. ダウンロードしたファイルをダブルクリック
  4. 基本的に「Next」を押し続ければOK

#### ③ VSCode（ブイエスコード）をインストール

**VSCodeとは？**  
コードを書くためのエディター（メモ帳の高機能版）です。

**インストール方法**:

1. https://code.visualstudio.com にアクセス
2. 「Download」ボタンをクリック
3. ダウンロードしたファイルをダブルクリック
4. 指示に従ってインストール

**推奨設定**（インストール後）:

VSCodeを開いて、左側の「拡張機能」アイコン（四角が4つ並んでいるアイコン）をクリックし、以下を検索してインストール:

- `ES7+ React/Redux/React-Native snippets`
- `Tailwind CSS IntelliSense`
- `Prettier - Code formatter`

1. https://github.com にアクセス
2. 「Sign up」をクリック
3. メールアドレス、パスワードを入力
4. ユーザー名を決める（例: `sota-narahara`）

**💡 ポイント**: ユーザー名は後から変更できません。わかりやすい名前にしましょう。

4. GitHubの認証画面で「Authorize Vercel」をクリック

**💡 ポイント**: GitHubアカウントと連携することで、後の作業が簡単になります。

## Step 2: サイトを作る（開発）

### 2-1. ターミナル/コマンドプロンプトを開く

- **Macの場合**: 
  - Spotlight検索（Command + Space）で「ターミナル」と入力

- **Windowsの場合**: 
  - スタートメニューで「cmd」と検索して「コマンドプロンプト」を開く

### 2-2. プロジェクトフォルダに移動

ターミナル/コマンドプロンプトで以下を入力（1行ずつ）:

```bash
cd /Users/sota.narahara/01_Sota.N/10_Projects/My-Profile-Site
```

**💡 説明**: `cd`は「チェンジ・ディレクトリ」の略。指定したフォルダに移動するコマンドです。

### 2-3. Next.jsプロジェクトを作成

以下のコマンドを実行:

```bash
npx create-next-app@latest . --typescript --tailwind --app
```

**実行後の質問への回答**:
- `Ok to proceed?` → **y** (Enterを押す)
- その他の質問は全て **Enter** を押して進める

**💡 説明**: これで、サイトの土台（テンプレート）が自動的に作成されます。

### 2-4. 追加のパッケージをインストール

以下のコマンドを実行:

```bash
npm install framer-motion react-hook-form zod @hookform/resolvers
```

**💡 説明**: サイトに必要な追加機能をインストールしています。

### 2-5. 開発サーバーを起動して確認

以下のコマンドを実行:

```bash
npm run dev
```

**成功すると**:
```
- Local:   http://localhost:3000
```
のような表示が出ます。

ブラウザで `http://localhost:3000` を開くと、サイトのテンプレートが表示されます。

**💡 ポイント**: これはまだ「あなたのパソコンの中だけ」で動いています。他の人は見られません。

### 2-6. コードを書いていく（本格的な開発）

ここから、`01_技術設計書.md` と `02_コンテンツデータ.md` を参考に、実際にコードを書いていきます。

**非エンジニアの方へ**:

この部分は、プログラミングの知識が必要になります。以下のいずれかの方法をお勧めします：

#### オプション1: 自分で学びながら作る

  - Next.js公式チュートリアル: https://nextjs.org/learn

## Step 3: GitHubに保存する

サイトが完成したら（または途中でも）、GitHubに保存します。

### 3-1. GitHubでリポジトリを作成

1. https://github.com にログイン
2. 右上の「+」マークをクリック → 「New repository」
3. 以下を入力:
   - **Repository name**: `my-portfolio-site`（お好きな名前でOK）
   - **Description**: `Sota.N Portfolio Site`（説明）
   - **Public** を選択（公開する場合）
   - 「Create repository」をクリック

### 3-2. ローカルのコードをGitHubにアップロード

ターミナル/コマンドプロンプトで、プロジェクトフォルダに移動した状態で以下を実行:

```bash
# Gitの初期化
git init

# 全てのファイルを追加
git add .

# コミット（変更を記録）
git commit -m "Initial commit"

# GitHubのリポジトリと紐付け（URLは自分のものに変更）
git remote add origin https://github.com/あなたのユーザー名/my-portfolio-site.git

# ブランチ名をmainに変更
git branch -M main

# GitHubにアップロード
git push -u origin main
```

**💡 説明**:
- `git add .` : 全ての変更をステージング（次のコミットに含める準備）
- `git commit -m "..."` : 変更を記録（メッセージ付き）
- `git push` : GitHubにアップロード

**初回のみ**: GitHubのユーザー名とパスワード（またはトークン）を聞かれます。

---

## Step 4: Vercelで公開する

### 4-1. Vercelでプロジェクトをインポート

1. https://vercel.com にログイン
2. 「Add New...」 → 「Project」をクリック
3. 「Import Git Repository」の下に、先ほど作成したGitHubリポジトリが表示されるはずです
4. 「Import」をクリック

### 4-2. プロジェクト設定

- **Project Name**: `my-portfolio-site`（自動で入力されます）
- **Framework Preset**: `Next.js`（自動で検出されます）
- **Root Directory**: `./`（そのまま）
- **Build and Output Settings**: そのままでOK

「Deploy」ボタンをクリック

### 4-3. デプロイ完了を待つ

数分待つと、デプロイが完了します。

**完成！🎉**

以下のようなURLが表示されます：
```
https://my-portfolio-site-xxx.vercel.app
```

このURLをブラウザで開くと、あなたのサイトが世界中から見られるようになっています！

## 🔄 更新する方法（サイトの内容を変更したい時）

### ローカルで変更 → GitHubにアップロード → 自動でVercelに反映

```bash
# 変更を保存
git add .
git commit -m "Update: プロフィール写真を追加"
git push origin main
```

Vercelは、GitHubにプッシュすると**自動的に**サイトを更新してくれます。

**所要時間**: 1〜2分で更新完了

`https://sota-n.com` のような独自ドメインを使いたい場合：

### 5-1. ドメインを取得

以下のいずれかのサービスでドメインを購入:

- **ムームードメイン**: https://muumuu-domain.com
- **Googleドメイン**: https://domains.google

**費用**: 年間1,000円〜3,000円程度（.comの場合）

### 5-2. Vercelでドメインを設定

1. Vercelのプロジェクト画面で「Settings」→「Domains」
2. 取得したドメイン名を入力（例: `sota-n.com`）
3. 「Add」をクリック
4. 表示された設定（AレコードまたはCNAME）をコピー

### 5-3. ドメインサービス側で設定

1. お名前.comなどのドメイン管理画面にログイン
2. DNS設定画面を開く
3. Vercelで表示された設定を入力
4. 保存

**反映時間**: 数時間〜24時間

## ❓ よくある質問

### Q1. お金はかかりますか？
**A**: 基本的に無料です。

- **無料で使えるもの**: GitHub、Vercel（個人サイトは無料プラン内）
- **有料になるもの**: 独自ドメイン（年間1,000円〜、任意）

### Q2. プログラミングの知識がないと無理ですか？
**A**: Cursorを使えば、AIと一緒に開発できます。

ただし、以下の基礎知識があると作業がスムーズです：
- HTML/CSSの基礎
- ターミナル/コマンドプロンプトの基本操作

**学習リソース**: 
- Progate（プロゲート）: https://prog-8.com
- ドットインストール: https://dotinstall.com

### Q3. 途中で挫折したらどうすればいいですか？
**A**: 以下の選択肢があります：

1. **Cursorに相談**: 「この部分がわからない」と具体的に聞く
2. **開発者に依頼**: ココナラ、ランサーズで外注
3. **一旦保留**: スキルアップしてから再挑戦

### Q4. サイトを非公開にできますか？
**A**: はい、できます。

- **Vercelの場合**: Settings → General → Deployment Protection で設定
- **GitHubの場合**: Settings → Change visibility → Private

### Q5. スマホでも見られますか？
**A**: はい、設計書の通りに作れば、自動的にレスポンシブ対応になります。
